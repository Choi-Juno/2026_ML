# 초기 설계 & 1차 실행

이 프로젝트의 목적은 사용자가 현재 이용 중인 구독 서비스에 대해 "이 구독을 계속 유지할 필요가 있을까?"를 데이터 기반으로 판단할 수 있도록 돕는 것입니다. 사용 패턴, 비용 대비 이용 효율, 불편 신호를 바탕으로 각 구독을 `유지 후보`와 `해지 후보`로 분류하는 초기 기준 모델을 설계했습니다.

---

## 1. 문제 정의서

### 해결할 문제
구독 서비스는 자동 결제로 이어지기 때문에, 사용 빈도가 낮아져도 관성적으로 유지되는 경우가 많습니다. 이 프로젝트는 사용자가 실제로 충분히 활용하고 있는지, 비용 대비 만족도가 낮아진 것은 아닌지, 불편 신호가 누적되고 있는지를 분석해 "계속 유지할 만한 구독인가"를 판단하는 문제를 다룹니다.

### 예측 목표
다음 결제 시점 기준으로 해당 구독이 사용자에게 계속 필요한지 여부를 이진 분류로 예측합니다.

- `0`: 해지 후보, 유지 필요성이 낮은 구독
- `1`: 유지 후보, 계속 사용할 가능성이 높은 구독

모델의 결과는 단순한 해지 예측이 아니라, 사용자 관점에서 정리 우선순위를 제안하는 의사결정 보조 도구로 활용하는 것을 목표로 합니다.

---

## 2. 데이터 설명

- 데이터 파일: `mock_data.csv`
- 데이터 규모: `10,000`건
- 데이터 성격: 초기 모델 검증을 위한 합성 데이터

### 주요 변수
| 변수명 | 설명 | 사용자 관점 해석 |
|---|---|---|
| `Monthly_Fee` | 월 구독료 | 비용 부담이 큰 구독인지 판단 |
| `Access_Days` | 월 접속 일수 | 실제로 자주 사용하는 서비스인지 판단 |
| `Usage_Time` | 월 누적 이용 시간 | 구독 활용도가 충분한지 판단 |
| `Content_Count` | 콘텐츠 소비량 | 이용 깊이와 몰입도 보조 판단 |
| `Customer_Inquiry` | 고객센터 문의 횟수 | 불편 경험이 누적되는지 확인 |
| `Inquiry_Rate` | 문의율 | 접속 대비 불만 비율 |
| `Value_Score` | 가성비 점수 | 비용 대비 실제 사용 효율 |

---

## 3. 기본 모델 실행

초기 기준 모델은 `Logistic Regression`으로 설정했습니다. 기준선을 먼저 확보하기 위해 데이터를 학습용과 테스트용으로 `8:2` 분리했고, 수치형 변수에는 `StandardScaler`를 적용했습니다. 이후 로지스틱 회귀 모델로 각 구독이 `유지 후보`인지 `해지 후보`인지 예측하도록 구성했습니다.

- 입력 변수: 이용 빈도, 사용 시간, 소비량, 문의 횟수, 파생 변수
- 타깃 변수: `0=해지 후보`, `1=유지 후보`
- 실행 환경: `churn_prediction.ipynb`

---

## 4. 성능 지표 해석

초기 기준 모델의 평가 결과는 다음과 같습니다.

- `Accuracy`: `0.9955`
- `Precision` (`0` 기준): `0.9951`
- `Recall` (`0` 기준): `0.9830`
- `F1-Score` (`0` 기준): `0.9890`
- `ROC-AUC`: `0.9997`

해지 후보를 찾는 관점에서 가장 중요한 지표는 `Recall`입니다. 실제로 유지 필요성이 낮은 구독을 놓치지 않아야 사용자가 불필요한 구독을 정리하는 데 도움이 되기 때문입니다. 현재 결과는 매우 높게 나타났지만, 이는 합성 데이터의 패턴이 비교적 뚜렷하게 설계된 영향도 함께 반영된 수치로 해석해야 합니다.

---

## 5. 현재 한계 정리

### 한계
1. 현재 데이터는 실제 사용자 로그가 아닌 합성 데이터이므로 현실의 복잡한 사용 패턴을 충분히 반영하지 못합니다.
2. 로지스틱 회귀는 선형 기반 모델이어서 구독 유지 판단에 영향을 주는 비선형 상호작용을 충분히 포착하기 어렵습니다.
3. 현재 변수만으로는 콘텐츠 만족도, 대체 서비스 존재 여부, 시즌성 사용 패턴 같은 실제 의사결정 요인을 반영하기 어렵습니다.

### 개선 방향
1. 실제 서비스 로그 또는 더 현실적인 시뮬레이션 데이터를 확보합니다.
2. `Decision Tree`, `Random Forest`, `XGBoost` 등 비선형 모델과 성능을 비교합니다.
3. 임계값 조정과 하이퍼파라미터 튜닝으로 해지 후보 탐지 성능을 고도화합니다.
