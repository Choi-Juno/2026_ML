# 초기 설계 & 1차 실행 프로젝트 개요

이 문서는 사용자 관점에서 "이 구독을 계속 유지할 필요가 있을까?"를 판단하기 위한 churn 분류 프로젝트의 핵심 내용을 한눈에 보기 쉽게 정리한 요약본이다.

---

## 1. 문제 정의서

### 해결할 문제
사용자는 여러 구독 서비스를 동시에 이용하지만, 실제로는 거의 사용하지 않는 구독도 자동 결제로 계속 유지하는 경우가 많다.  
따라서 이용 패턴과 비용 대비 효율을 바탕으로, 현재 유지 중인 구독이 정말 필요한지 판단할 수 있는 기준이 필요하다.

### 예측 목표
- 각 구독을 `해지 후보`와 `유지 후보`로 이진 분류
- `0`: 해지 후보, 유지 필요성이 낮은 구독
- `1`: 유지 후보, 계속 유지할 가능성이 높은 구독
- 사용자가 구독 정리 우선순위를 정할 수 있도록 지원

## 2. 주요 변수
| 변수명 | 의미 |
|---|---|
| `Monthly_Fee` | 월 구독료 |
| `Access_Days` | 월 접속 일수 |
| `Usage_Time` | 월 누적 이용 시간 |
| `Content_Count` | 콘텐츠 소비량 |
| `Customer_Inquiry` | 고객센터 문의 횟수 |
| `Inquiry_Rate` | 접속 대비 문의 비율 |
| `Value_Score` | 비용 대비 사용 효율 |

## 3. 기본 모델 실행
- 데이터: `mock_data.csv`, 총 `10,000`건
- 모델: `Logistic Regression`
- 분할: 학습 `80%`, 테스트 `20%`
- 전처리: `StandardScaler` 적용
- 목적: 초기 기준선 확보

## 4. 성능 지표 해석
- `Accuracy`: `0.9955`
- `Precision` (`0` 기준): `0.9951`
- `Recall` (`0` 기준): `0.9830`
- `F1-Score` (`0` 기준): `0.9890`
- `ROC-AUC`: `0.9997`

현재 기준 모델은 해지 후보를 매우 잘 찾아내는 것으로 나타났지만, 이는 합성 데이터의 규칙성이 강한 영향도 함께 반영된 결과다.

## 5. 현재 한계 정리
1. 실제 사용자 로그가 아닌 합성 데이터 기반 결과다.
2. 로지스틱 회귀만으로는 복잡한 비선형 패턴을 충분히 반영하기 어렵다.
3. 만족도, 대체 서비스 유무 등 실제 유지 판단 요인이 아직 포함되지 않았다.
